# Word

![](images/word.png)

Откройте **4s-файл**, который у вас получился после [парсинга](../4s). После нажатия кнопки «Запустить» в той же папке, что и 4s-файл, появится файл в формате docx.

## Спойлеры

Полезны для случаев, когда надо отправить кому-то файл на тестирование. По умолчанию спойлеры отключены, они бывают трёх видов:

- `whiten` — спойлеры забелением. Неудобны при тестировании с мобильного.
- `pagebreak` — спойлеры переносом страницы. Гугл-докс вырезает переносы страницы на мобильном, поэтому могут быть тоже не идеальны.
- `dots` — спойлеры отбивкой точками. Выглядит не очень эстетично, зато работает, в том числе в гугл-докс с мобильного.

## Версия для экрана {: #screen }

Версия, в которой удалены ударения и указания ведущему в квадратных скобках, а также внесены изменения из [тега `screen`](../4s#screen). Нужна для онлайнов.

Существует в двух видах:

- `replace_all` — во всех вопросах вместо версии для ведущего выводит версию для экрана. При выборе этого варианта к имени файла будет добавлен фрагмент `_screen`.
- `add_versions` — каждый вопрос сначала приводится в версии для ведущего, а потом в версии для экрана. Удобно для случаев, когда не используются сервисы типа [OpenQuiz](https://open-quiz.com), а ведущий скидывает текст вопроса команде в чат. При выборе этого варианта к имени файла будет добавлен фрагмент `_screen_versions`.

Квадратные скобки можно эскейпить, если написать перед ними слеш, `\[вот так\]` — тогда они не будут удаляться.
